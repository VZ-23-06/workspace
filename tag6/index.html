Was ist Git & GitHub?

Was ist Git?

Git ist ein weit verbreitetes verteiltes Versionskontrollsystem. Es wird hauptsächlich von Entwicklern:innen verwendet, um die Versionsgeschichte von Quellcode und anderen Dateien zu verfolgen, Änderungen zu verwalten und die Zusammenarbeit in Softwareentwicklungsprojekten zu erleichtern.

  1. Versionskontrolle und Sicherung von Daten: Git ermöglicht die effiziente Verfolgung von Änderungen am Quellcode und anderen Dateien.
  2. Commits:Ein Commit ist eine Gruppe von Änderungen, die zu einem Zeitpunkt in die Versionsgeschichte aufgenommen werden.
  3. Branches: Ein Branch ist eine separate Entwicklungslinie, die von der Hauptentwicklungslinie (normalerweise als "Main" bezeichnet) abzweigen kann.
  4. Merge und Pull Requests: Nachdem Änderungen in einem Branch entwickelt wurden, können sie durch Zusammenführen (Merge) in den Hauptcode integriert werden. In Open-Source-Projekten oder Kollaborationen können wir sogenannte Pull Requests erstellen, um Änderungen in den Hauptcode vorzuschlagen. Andere Entwickler überprüfen diese Änderungen und genehmigen oder lehnen sie ab.
  5. Konfliktlösung:Wenn zwei oder mehr Entwickler gleichzeitig Änderungen an derselben Datei vornehmen, kann es zu Konflikten kommen.
  6. Remote-Repositories: Wir können unsere Projekte auf remote gehosteten Git-Servern speichern, um die Zusammenarbeit zu ermöglichen.

Was ist GitHub?

GitHub ist eine webbasierte Plattform, die hauptsächlich von Entwicklern und Softwareentwicklungsprojekten genutzt wird, um Code zu hosten, zu verwalten und gemeinsam daran zu arbeiten. Es bietet eine Umgebung, in der Entwickler ihre Programmierprojekte speichern, Versionskontrolle durchführen, Änderungen nachverfolgen, Code überprüfen und zusammenarbeiten können.

  1. Repositories: Projekte können in sogenannten Repositories gespeichert werden. Hier werden der Quellcode und andere Dateien für ein Projekt aufbewahrt.
  2. Versionskontrolle: GitHub verwendet das Versionskontrollsystem Git. Dies ermöglicht es uns, Änderungen am Quellcode zu verfolgen, zu vergleichen und zu verwalten.
  3. Branches:Branches: Wir können sogenannte Branches erstellen, um verschiedene Versionen des Codes zu entwickeln. Dies ermöglicht es mehreren Personen, gleichzeitig an verschiedenen Teilen des Projekts zu arbeiten, ohne sich gegenseitig zu beeinträchtigen.
  4. Pull Requests:Pull Requests: Wenn Entwickler:innen Änderungen in ein Repository einbringen möchten, können sie einen sogenannten Pull Request (PR) erstellen. Andere Entwickler können dann den vorgeschlagenen Code überprüfen, Kommentare abgeben und die Änderungen genehmigen oder ablehnen, bevor sie in den Hauptcode eingefügt werden.
